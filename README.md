# go-codeact-agent

A CLI-based AI coding agent built in Go that performs automated code reviews using a **Hybrid CodeAct pattern**. 

Unlike standard agents that rely purely on pre-defined tools, this agent uses a hybrid approach:
- **Standard Tools**: High-performance Go functions for common tasks (git diff, file reading) to accelerate the workflow.
- **CodeAct (PowerShell)**: Dynamic script generation and execution for complex exploration and environment interaction.

## Key Features

- **🚀 Hybrid Execution**: Combines the speed of native tool-calling with the flexibility of LLM-generated scripts.
- **🛡️ Safety First**: Optional human-in-the-loop approval before any script is executed locally.
- **🔍 Reliable Debugging**: Captures partial output on timeouts and extracts specific exit codes for better LLM self-correction.
- **📉 Context Optimized**: Smart output truncation (head/tail) to keep conversation history concise.
- **📡 Smart Rate Limiting**: Intelligent API backoff handling (429 errors) with exponential retry logic.

## How It Works

1. You provide a prompt describing what you want reviewed.
2. The agent sends it to the LLM along with available tools (git diff, file reading).
3. The LLM chooses between **Native Tools** (for speed/consistency) or **PowerShell Scripts** (for custom exploration).
4. Scripts are reviewed by the user (if enabled) and executed locally.
5. The loop continues until the LLM outputs `[DONE]`, delivering a structured Markdown review.

## Prerequisites

- [Go 1.22+](https://go.dev/dl/)
- [Git](https://git-scm.com/)
- PowerShell (comes pre-installed on Windows)
- A [Groq API key](https://console.groq.com/keys)

## Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/lbalbas/go-codeact-agent.git
   cd go-codeact-agent
   ```

2. **Create your `.env` file**
   ```bash
   cp .env.example .env
   ```
   Then edit `.env` and add your API keys:
   ```env
   GROQ_API_KEY="your-groq-api-key-here"
   ```

3. **Install dependencies**
   ```bash
   go mod download
   ```

## Usage

### Local Usage
Run the agent from the project root:
```bash
go run ./cmd/go-agent
```

### Global Installation (Recommended)
You can install the agent as a global CLI tool to use it in any project:

1. **Install the binary**:
   ```bash
   go install ./cmd/go-agent
   ```

2. **Set up Global Configuration**:
   Copy your `.env` file to your home directory so the agent can find your API key from anywhere:
   ```powershell
   # Windows
   copy .env $HOME\.go-agent.env
   ```

3. **Run from any project**:
   Open a terminal in the project you want to review and run:
   ```bash
   go-agent
   ```

## Adding New Tools

To add a new tool the LLM can call:

1. Create a handler function that accepts a JSON arguments string and returns a string result.
2. Register the tool's schema in the tool configuration.
3. Map the tool name to its handler in the tool registry.

## Security Note

> **⚠️ This agent executes PowerShell scripts generated by an LLM on your local machine.** By default, the agent **requires manual user approval** before running any generated script to ensure safety. Use it in controlled environments and never run it with elevated privileges.
